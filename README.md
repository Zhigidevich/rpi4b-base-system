# rpi4b-base-system
Загрузочная флешка для Raspberry PI 4B+

rpi-base-system/

├── scripts/              # Bash-скрипты для быстрой настройки (Этапы 1-4)

│   ├── 01_security.sh    # SSH, ключи, отключение паролей

│   ├── 02_system.sh      # Watchdog, CPU performance, tmpfs (логи)

│   ├── 03_network.sh     # Static IP, отключение энергосбережения Wi-Fi

│   └── 04_peripherals.sh # I2C, UART, SPI, CAN-интерфейс

├── configs/              # Шаблоны конфигов (которые вы будете копировать)

│   ├── config.txt        # Для активации шин RPi

│   ├── cmdline.txt

│   └── sshd_config

├── docs/                 # Документация по железу

└── README.md             # Инструкция: "Как развернуть базу на чистой Ubuntu 26.04"

Категория: Безопасность (Security)

[ ] Настройка SSH-авторизации только по ключам, отключение паролей.

[ ] Ограничение прав доступа и базовая настройка Firewall.

[ ] Базовая настройка Git (Global Config) для системы.

Категория: Производительность (System Optimization)

[ ] Оптимизация питания: фиксация CPU в режиме performance.

[ ] Активация аппаратного Watchdog.

[ ] Защита SD-карты от износа: перенос /var/log в RAM (tmpfs).

[ ] Настройка GPU Memory Split для задач с видео.

Категория: Сеть и периферия (Hardware & Connectivity)

[ ] Активация аппаратных шин: I2C, UART, SPI.

[ ] Конфигурация интерфейса CAN (can0).

Категория: Защищённое подключение из Ethernet (VPN Layer)

[ ] Отключение энергосбережения Wi-Fi (NetworkManager).

[ ] Настройка статического IP.

[ ] Установка пакета (Tailscale/WireGuard).

[ ] Генерация ключей и привязка к вашему личному VPN-узлу.
Настройка tailscaled в режиме exit-node или relay

[ ] Настройка systemd-сервиса, чтобы VPN поднимался до запуска основных ROS2-нод (важно, чтобы робот сразу был в сети).

[ ] Маршрутизация и DNS, чтобы трафик управления (DDS) корректно маршрутизировался через VPN-туннель.

[ ] Настройка systemd-сервиса, чтобы VPN поднимался до запуска основных ROS2-нод (важно, чтобы робот сразу был в сети).

[ ] Отказоустойчивость (Watchdog для сети): Скрипт мониторинга соединения: если VPN-туннель «завис» или сеть пропала, скрипт должен автоматически перезапускать интерфейс или весь сетевой стек (настройка netplan или nmcli).

[ ] Агент мониторинга соединения (Connectivity Watchdog): Скрипт, который пингует не только локальный шлюз, но и внешний публичный IP (например, 8.8.8.8) через VPN-интерфейс. Если связи нет — перезагрузка VPN-сервиса.

[ ] Fallback-механизм (Аварийный доступ): Если основной VPN «лежит», нужно иметь альтернативу. Например, настроить SSH-туннель с обратным пробросом портов (Reverse SSH Tunnel) на ваш публичный сервер.

[ ] Безопасность фаервола: Настройка ufw (Uncomplicated Firewall) для разрешения входящих соединений только через VPN-интерфейс и локальную сеть.
